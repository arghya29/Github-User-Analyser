# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in GitHub User Analyzer, please email us privately instead of using the issue tracker. This allows us to fix the issue before it's publicly disclosed.

**DO NOT** open a public issue for security vulnerabilities.

### How to Report

Please include:
- A description of the vulnerability
- Steps to reproduce
- The potential impact
- Any possible fixes or workarounds

We will acknowledge your report within 48 hours and provide updates regularly.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |

## Security Best Practices

When using GitHub User Analyzer:

- Keep Node.js and dependencies up to date
- Never hardcode sensitive information (API keys, tokens)
- Use environment variables for secrets
- Review code before deploying to production
- Report any suspicious activity immediately

## Thanks

We appreciate your responsible disclosure and commitment to keeping our project secure!
